1. Quickstart & Local Setup

Follow these steps to run SHIELD_IDS on your local development machine:

Step 1: Clone Repository & Create Virtual Environment
git clone https://github.com/Krishnan533/AI-Powered-Cyber-Threat-Detection-System.git cd mini-project python -m venv venv # On Windows: .\venv\Scripts\activate # On Linux/macOS: source venv/bin/activate
Step 2: Install Dependencies
pip install -r requirements.txt
Step 3: Launch Local Application
python wsgi.py

The application will boot on http://localhost:5000 with default SQLite database setup.

2. Render Deployment Guide

Configured for seamless deployment on Render Web Services:

Setting Configured Value
Runtime Environment Python 3.11.9
Build Command pip install --upgrade pip && pip install --no-cache-dir -r requirements.txt
Start Command gunicorn -c gunicorn.conf.py wsgi:app
Database Engine PostgreSQL (Render PostgreSQL Managed Instance)

3. Environment Variables

FLASK_ENV=production PYTHONPATH=. PYTHON_VERSION=3.11.9 SECRET_KEY=your_generated_secret_key DATABASE_URL=postgresql://user:pass@host:5432/dbname SNIFFER_SIMULATION=TRUE SEND_EMAIL_NOTIFICATIONS=TRUE ADMIN_EMAIL=admin-alerts@example.com

4. REST API Specification

POST /api/auth/register

Registers a new user account with role assignment (Viewer/Analyst/Admin).

POST /api/auth/login

Authenticates username and password, returning JWT bearer token and setting session cookies.

GET /api/dashboard/stats

Returns system-wide packet counts, threat severities, active firewall rules, and hourly telemetry.

GET /api/threats

Fetches recorded threat events with filtering by severity, status, and IP address.

POST /api/blocked-ips

Manually creates a firewall containment rule to block a malicious IP.

5. Troubleshooting & FAQ

Render web service containers run as unprivileged non-root users. Live network socket sniffing requires root privileges (CAP_NET_RAW). Setting SNIFFER_SIMULATION=TRUE allows the system to generate mock traffic, DDoS spikes, port scans, and AI anomalies cleanly for SOC telemetry without socket errors.

The application inspects DATABASE_URL at startup. If present (PostgreSQL on Render), it automatically normalizes postgres:// to postgresql://. If absent, it automatically resolves absolute local paths for SQLite in instance/network_monitor.db.
